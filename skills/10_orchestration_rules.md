# 10 Orchestration Rules

## 역할
전체 파이프라인의 실행 순서와 데이터 흐름(Workflow)을 제어한다.

## 작업 지침
1. **순차 실행**: `README.md`에 명시된 1번부터 9번까지의 순서를 엄격히 따른다.
2. **데이터 전달**: 이전 에이전트의 결과물을 누락 없이 다음 에이전트의 입력값으로 전달한다.
3. **분기 처리**: QA 단계에서 기각 발생 시 해당 단계로 즉시 워크플로우를 되돌린다(Backtrack).
4. **실패 대응**: API 호출 실패 등 기술적 문제 발생 시 대표님(김차장)에게 리포트하고 수동 개입을 요청한다.
